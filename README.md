# Requirements Definition – Web-Based Calculator

## 1. Purpose
The purpose of this system is to develop a simple interactive calculator that runs in a web browser. The application allows users to input numbers, select an arithmetic operation, and receive immediate results. It demonstrates fundamental programming concepts such as user input, selection, validation, and output.

---

## 2. Target Users
- Students learning basic programming concepts  
- Beginner users needing a simple calculator  
- Software Engineering students (primary audience)  

---

## 3. Functional Requirements
The system must:

1. Allow the user to enter their name  
2. Allow the user to input two numerical values  
3. Provide a selection of operations:
   - Addition  
   - Subtraction  
   - Multiplication  
   - Division  
4. Perform the selected calculation when the user clicks a button  
5. Display the result clearly on the screen  
6. Display a personalised message including the user’s name  
7. Detect and handle invalid inputs (e.g., non-numeric values)  
8. Prevent division by zero and display an appropriate error message  
9. Allow repeated use without needing to reload the page  

---

## 4. Non-Functional Requirements
The system should:

- Be easy to use with a clear and simple interface  
- Respond quickly to user actions  
- Be compatible with standard web browsers (e.g., Chrome, Edge)  
- Display readable text and organised layout  
- Be reliable and free of major errors  
- Require minimal system resources  

---

## 5. User Interface Requirements
- Input fields for:
  - User name  
  - First number  
  - Second number  
- A dropdown menu for selecting the operation  
- A clearly labelled "Calculate" button  
- An output area to display results or error messages  
- Logical layout with clear labels and spacing  

---

## 6. System Constraints
- Must be developed using HTML and JavaScript  
- Must run entirely within a web browser  
- No external libraries required  
- Limited to basic arithmetic operations  
- Developed within the time constraints of a school project  

---

## 7. Assumptions
- Users have basic knowledge of how to use a web page  
- Users can enter valid numerical input  
- The system will be used on a device with a modern web browser  

---

## 8. Success Criteria
The system will be considered successful if:

- Users can perform calculations easily and accurately  
- Errors are handled correctly without crashing the program  
- The interface is intuitive and easy to navigate  
- The application runs smoothly in a web browser
